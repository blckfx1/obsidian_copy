1. Core Features (Minimum Viable Product)
Feature	Description
Dark design	Dark background, light text, minimal UI
Three‑pane layout	Three resizable/scrollable note editors open at once
Note manager	Create, rename, delete notes – left sidebar lists all note titles
Wiki‑links	[[Note Title]] syntax – clickable, automatically creates/linked notes
Search	Full‑text search over note titles + content; results clickable
Persistence	All notes saved automatically in browser’s IndexedDB
Many notes	IndexedDB can handle thousands of notes (>10k is fine)
2. Technology Stack (Simple & No Build Step)
HTML5 – structure

CSS3 – dark theme, flex/grid for panes

Vanilla JavaScript (ES6) – no frameworks, easier to customise

IndexedDB (via idb wrapper or native API) – local database

LocalStorage – only for user preferences (e.g. last opened notes)

Why no React/Vue? You wanted “from scratch” – vanilla gives you total control and is easier to debug for a small project.

3. Data Model (IndexedDB Schema)
We’ll create one object store: notes

Field	Type	Description
id	string	Unique ID (e.g. timestamp + slug)
title	string	Note title (used in [[title]])
content	string	Markdown/plain text
updatedAt	number	Timestamp for sorting
Indexes: title (unique), updatedAt

Phase 2: IndexedDB Setup & Note CRUD (2–3 hours)
Write db.js module to open database, create store, and handle promises.

Implement functions:

getAllNotes() → array of {id, title, updatedAt}

saveNote(note) → upsert by id

deleteNote(id)

getNote(id)

Auto‑save on textarea input events (debounced).

When you create a new note, assign a unique ID and default title “Untitled”.

Phase 3: Note List & Three Pane Management (2 hours)
On page load, load all notes into left sidebar (sorted by updatedAt).

Each note in sidebar has: title, edit icon, delete icon.

User can click a note title → open it in the first empty pane (or replace the least recently used pane).

Keep track of which pane is showing which note ID.

When you edit a pane’s textarea, update that note’s content in DB.

Phase 4: Wiki‑Link Parsing & Navigation (2 hours)
Detect [[...]] inside note content.

When a pane’s content changes, scan for links and replace them with <span class="wikilink">[[...]]</span> + click handler.

On click, extract the link target title, search for a note with that title, and open it in another pane (or create it if not exists).

Create‑on‑click: if note doesn’t exist, create a new empty note with that title and open it.

Implementation tip: store note titles in Map for quick lookup. When rendering a pane, convert text with regex:
content.replace(/\[\[(.*?)\]\]/g, '<span class="wikilink" data-title="$1">[[$1]]</span>').

Phase 5: Search Engine (2 hours)
Add search input in header.

On each keystroke, filter notes using two strategies:

Quick: search note titles for substring.

Full‑text: iterate over all notes and check content.includes(query).
(For many notes, add a simple index later, but with 1000 notes modern browsers handle this fine.)

Display results as a dropdown list under search bar.

Click a result → open that note in a pane.

Phase 6: Polish & Edge Cases (1–2 hours)
Debounce DB writes (500ms) to reduce I/O.

Delete confirmation before removing a note.

When you rename a note (by editing its title in the pane header), update all [[oldTitle]] links to [[newTitle]] in other notes (optional – start with simple update).

Ensure you can’t open the same note in two panes at once (disable or sync).

Add keyboard shortcuts: Ctrl+N new note, Ctrl+S save (auto-save anyway).

Responsive: collapse panes on small screens (optional).

6. Key Code Snippets (to get you started)
IndexedDB wrapper (simplified)
javascript
// db.js
const DB_NAME = 'ObsidianClone';
const STORE_NAME = 'notes';

let db = null;

export async function initDB() {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open(DB_NAME, 1);
    request.onupgradeneeded = (e) => {
      const db = e.target.result;
      if (!db.objectStoreNames.contains(STORE_NAME)) {
        const store = db.createObjectStore(STORE_NAME, { keyPath: 'id' });
        store.createIndex('title', 'title', { unique: true });
        store.createIndex('updatedAt', 'updatedAt');
      }
    };
    request.onsuccess = () => { db = request.result; resolve(db); };
    request.onerror = () => reject(request.error);
  });
}

export async function getAllNotes() {
  const tx = db.transaction(STORE_NAME, 'readonly');
  return tx.objectStore(STORE_NAME).getAll();
}

export async function saveNote(note) {
  note.updatedAt = Date.now();
  const tx = db.transaction(STORE_NAME, 'readwrite');
  return tx.objectStore(STORE_NAME).put(note);
}

function renderLinks(content, targetPaneId) {
  const linkedHTML = content.replace(/\[\[(.*?)\]\]/g, (match, title) => {
    return `<span class="wikilink" data-title="${escapeHtml(title)}">[[${escapeHtml(title)}]]</span>`;
  });
  document.getElementById(targetPaneId).innerHTML = linkedHTML;
  // attach click listeners to .wikilink
}
Search (naive full‑text)
javascript
async function searchNotes(query) {
  const all = await getAllNotes();
  return all.filter(n => n.title.includes(query) || n.content.includes(query));
}
